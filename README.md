# 予約予測シミュレーター（統合版）— Threads × Instagram

ホテル向けの営業ツールです。**Threadsの打数（フロー）・Instagramのフィード/リール（フロー＋フォロワー獲得）・フォロワー資産（ストック・複利）**の3エンジンを統合し、最大36ヶ月の予約数・売上の推移を試算します。

- **公開URL**：（下の「公開URL」を参照）
- **1ファイル完結**：`index.html` だけで動きます（サーバー・ビルド・外部ライブラリなし、グラフもSVG内製）。
- **設定はURLで共有可能**：画面の「この設定の共有リンクをコピー」を押すと、今の設定込みのURLがコピーされます。相手がそのURLを開くと同じ設定が再現されます。

## 計算モデル（さわらなくてOK）

```
月間予約 = Threadsフロー + IGフロー + IGストック
  Threadsフロー = imp × クリック率 × 到達率 × 予約CVR
  IGフロー      = imp × プロフィール率 × リンクタップ率 × 予約CVR
  IGストック    = フォロワー総数 × 月間予約率
フォロワー総数(今月) = 総数(先月) × (1 − 離脱率) + 月間新規フォロワー
```

フォロワーが毎月積み上がり、その分の予約（ストック）が**複利**で増えていくのがこのツールの肝です。

## 数値の「初期値」を直したいとき（非エンジニア向け）

直す場所は `index.html` の中の **`var F = [ ... ]` という一覧**です（だいたい 139行目あたり）。
各行は次の並びで、**行のいちばん最後の数字が「初期値」**です。

```
["項目id","グループ","ラベル","補足","型",最小,最大,刻み,初期値]
```

例：Threadsのアカウント数の初期値を 20 → 30 にしたい場合

```js
["th_accounts","th","アカウント数","","num",1,200,1,20],
                                              ↑ この最後の 20 を 30 に変える
```

よく触る項目の目安：

| 行の id | 意味 |
|---------|------|
| `th_accounts` / `th_posts` / `th_imp` | Threadsのアカウント数・投稿数・平均インプレッション |
| `th_ctr` / `th_reach` / `th_cvr` | Threadsのクリック率・到達率・予約CVR |
| `ig_reels` / `ig_feed` | Instagramのリール・フィードの月間インプレッション |
| `ig_profile` / `ig_link` / `ig_cvr` | IGのプロフィール率・リンクタップ率・予約CVR |
| `f_churn` / `f_rate` | フォロワーの月間離脱率・月間予約率 |
| `adr` | 平均予約単価（円） |
| `months` | シミュレーション期間（ヶ月、最大36） |

> 「最小・最大・刻み」（最後から2〜4番目の数字）を変えると、スライダーで動かせる範囲も変えられます。
> **配色・レイアウト・計算式は変えないでください**（営業資料としての見た目と数値の一貫性を保つため）。

## 変更を「再公開」する手順

GitHub Pages で公開しているので、**`main` ブランチに反映すると1〜2分で自動的に本番へ反映**されます。

### パターンA：GitHubのサイト上だけで直す（ターミナル不要・かんたん）

1. ブラウザで https://github.com/Ryotaro0416/booking-simulator-integrated を開く
2. `index.html` をクリック → 右上の鉛筆アイコン（Edit）
3. 数字や文言を直す
4. 右上の **Commit changes** を押す
5. 1〜2分後に公開URLへ反映（反映が遅いときは Mac で `Cmd+Shift+R` のスーパーリロード）

### パターンB：ターミナルが使える場合

```bash
cd ~/booking-simulator-integrated
git add -A
git commit -m "数値を更新"
git push
```

## 動作確認のしかた

- 公開URLを開き、スライダーや数値を動かすと、KPIカード・積み上げグラフ・内訳がリアルタイムに変わればOK。
- 「この設定の共有リンクをコピー」を押し、別タブにそのURLを貼って開くと、同じ設定が再現されればOK。
- 例：URLの末尾に `#f_rate=2&months=24` を付けて開くと「月間予約率=2%・期間=24ヶ月」になり、グラフが24ヶ月分に伸び、フォロワー資産（ストック）が複利で増えていきます。

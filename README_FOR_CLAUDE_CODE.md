## 1. これは何か

ホテル向けマーケティングの営業ツールです。**Threads × Instagram の予約予測シミュレーター（統合版）**。
3つのエンジンを月次でまわし、12ヶ月の予約数・売上の推移を試算します。

- **Threads（フロー・打数）**: インプレッション → クリック → 予約
- **Instagram フィード/リール（フロー＋フォロワー獲得）**: インプレッション → プロフィール → 予約、かつ新規フォロワーを供給
- **フォロワー資産（ストック・複利）**: フォロワー総数 × 月間予約率（毎月積み上がり複利で効く）

## 2. このフォルダの中身

- `index.html` … シミュレーター本体。**1ファイルで完結**（バックエンド・ビルド・外部ライブラリなし、素のHTML/CSS/JS）。グラフもSVGを内製しており、オフラインでも動きます。
- `README_FOR_CLAUDE_CODE.md` … この指示書。

## 3. ゴール（＝完成の定義）

1. **誰でもアクセスできる公開URL**で `index.html` が動く（PC・スマホ両対応）。
2. スライダーや数値を変えると結果（KPI・積み上げグラフ・内訳）がリアルタイム更新される（※実装済み）。
3. **設定した状態をURLで共有できる**（※実装済み）。共有された相手がそのURLを開くと、同じ設定が再現される。
4. 無料または極小コストで運用できる。
5. 依頼者が後から数値や前提を更新したくなったとき、**更新→再公開の手順がREADMEに書いてある**。

## 4. すでに実装済み（あなたは壊さないこと）

`index.html` には次がすでに入っています。**新規実装は不要、動作確認だけ**してください。

- 3エンジンの計算ロジック（フロー＋ストックの複利モデル）、KPIカード、エンジン別の積み上げ棒グラフ、最終月の内訳、客単価・期間（最大36ヶ月）の設定。
- **URL共有機能**：設定変更時に `location.hash` に全入力値を保存（キーは各入力の id、例 `#th_accounts=20&ig_reels=500000&f_rate=1&months=12`）。読込時にハッシュを反映し、「この設定の共有リンクをコピー」ボタンでクリップボードにコピー。→ ハッシュ方式なので**静的ホスティングのままで動きます**。
- **OGP メタ情報**（`og:title` / `og:description`）も設定済み。

> 検証手順：`index.html#f_rate=2&months=24` のようなURLで開き、月間予約率=2%・期間=24ヶ月が反映され、グラフが24ヶ月分に伸び、フォロワー資産（ストック）が複利で増えていけばOK。また「共有リンクをコピー」で得たURLを別タブで開き、同じ設定が再現されることを確認。

## 5. あなた（Claude Code）にやってほしいこと

優先度順。**1つ終わるごとに簡潔に報告**してください。

1. **Gitリポジトリ化**：このフォルダを git 初期化し、適切な `.gitignore` を置く。
2. **動作確認**：上記「検証手順」を実施し、計算・グラフ・URL共有・スマホ表示が問題ないことを確認（OGPは追加済みなので確認のみ）。
3. **公開（デプロイ）**：**GitHub Pages を第一候補**にデプロイ。
   - `gh` CLI でリポジトリ作成 → push → Pages 有効化 → 公開URL取得。
   - `gh` 未認証なら、**ここで一度だけ依頼者にブラウザ認証を依頼**（手順を日本語で提示）。
   - 代替：Netlify（`netlify deploy --prod`）または Cloudflare Pages（`wrangler pages deploy`）。どれか1つで良い。
4. **公開URLの報告**：最終的な公開URLを依頼者に伝える。スマホでも開けることを確認。
5. **READMEの更新**：依頼者向けに「数値の初期値を直す場所（`index.html` 内の `F = [...]` 配列の各行の最後の値が初期値）」「変更を再公開する手順（`git push` で自動反映 等）」を平易な日本語で追記。

## 7. 守ること（制約・トンマナ）

- **デザインを変えない**：配色（ペトロル `#0E3A42` × ブラス `#C9A24B` × アイボリー `#F6F2EA`）、日本語UI、レイアウトは維持。
- **計算ロジックを変えない**：3エンジンの式・初期値・各指標の定義はそのまま。
  - モデル：`月間予約 = Threadsフロー + IGフロー + IGストック`、`フォロワー総数(t) = 総数(t-1)×(1-離脱率) + 月間新規フォロワー`、`IGストック = フォロワー総数 × 月間予約率`。
- 余計なライブラリやビルド工程を増やさない（**素のHTML1枚**の手軽さを保つ）。
- 「数値は試算であり成果を保証しない」旨の注記は残す。
